VC · UX decision records

Design decisions

A set of decision records for the recurring interface questions in VC. Each one states what we do, why, and what the major design systems do, so a contested choice gets settled by pointing at the record rather than relitigating it in every review.

Status shows how settled each record is. Draft is being written, Adopted is the house rule, and Superseded has been replaced by a newer record.

Component usage

Layout
Adopted
Modal, full page, or side panel
Full page by default for real work; side panel for work beside the main view; modal only for a short blocking decision.
Standards · Forms
Draft
Form layout on wide artboards
Size inputs to their content, not the artboard; one capped column by default. A date or a code is never full width.
 Draft
Labels, descriptions, hints and placeholders
Every field gets a visible label. A placeholder is an optional example, never a substitute for the label.
