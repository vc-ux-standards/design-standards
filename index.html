Component usage

Adopted Stepper orientation Vertical rail by default; horizontal only for short, linear, page-swapping wizards.

Layout

Adopted Modal, full page, or side panel Full page by default for real work; side panel for work beside the main view; modal only for a short blocking decision.

Standards

Forms

Draft Form layout on wide artboards Size the layout to the task, not the artboard: a capped column for sequential forms, a grid for dense records; every input sized to its content. Draft Descriptions and placeholders Guidance that must survive typing — meaning, format, constraints — goes in the description above the input; the placeholder is only a faint, optional example.

Responsive

Draft Breakpoints Desktop only, on a 12-column grid: width follows content type — datagrids and charts use the width, reading and config canvases cap the measure; narrow widths met by zoom reflow, not a mobile layout.

Start with the question

Draft Choose a chart Select the mark from the comparison, trend, part-to-whole, correlation, or hierarchy question.

Common chart forms

Draft Bars and columns Use zero-based, value-sorted bars; choose orientation for label length and count. Draft Stacked bars and tooltips Show decision-critical values directly; tooltips enrich the whole-column target. Draft Line charts and trends Show change over time with labelled zooms, honest gaps, and limited series. Draft Dot plots and ranges Use position marks and labelled zooms when zero-based bars flatten meaningful differences.

Presentation and scale

Draft Numbers, units, and data states Format for scanning and distinguish no data, zero, suppression, and loading. Draft Charts at tile scale Use a number and a minimal mark; charts requiring axes belong in a panel. Draft Colour for data Give colour one meaning and never make it the sole encoding.

Trust and interpretation

Draft Reading charts honestly Prevent known distortions in scales, denominators, comparison, and time windows.
