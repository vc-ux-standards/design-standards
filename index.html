Design decisions

Decision records for the recurring interface questions in VC. Each states what we do, why, and what the major design systems do — so a contested choice is settled by pointing at the record, not relitigating it in review.

Status shows how settled each record is. Draft is being written, Adopted is the house rule, and Superseded has been replaced by a newer record.

Standards · Forms
Draft
Form layout on wide artboards
Size the layout to the task, not the artboard: a capped column for sequential forms, a grid for dense records; every input sized to its content.
 Draft
Descriptions and placeholders
Guidance that must survive typing — meaning, format, constraints — goes in the description above the input; the placeholder is only a faint, optional example.
Standards · Responsive
Draft
Breakpoints
Desktop only, on a 12-column grid: width follows content type — datagrids and charts use the width, reading and config canvases cap the measure; narrow widths met by zoom reflow, not a mobile layout.
Standards · Data visualization
Draft
Choosing a chart
Start from the question — comparison, trend, part-to-whole, correlation, hierarchy — then encode the deciding value as position or length, not angle, area, or colour. The entry point to the data-viz standards.
 Draft
Bars: orientation, order, and too many
Sort by value, start the axis at zero, go horizontal for long labels; past ~12 categories show the top-N and group the tail into "Other".
 Draft
Line charts and trends
Change over time is a line: slope carries the value, so a zoomed y-axis is fair if you label it. Default to rolling 12 months, mark the partial period, break the line over gaps, and split past ~4 series into small multiples.
 Draft
Dot plots and the honest zoom
When values cluster near the top and a zero-based bar flattens them, switch to a dot plot on a labelled zoomed axis — position can zoom where length can't. Dumbbells for two states, sorted by the gap.
 Draft
Colour for data
Colour is the weakest channel: spend it on one job — separate categories (cap ~6), show magnitude, or show deviation from a target. Never decoration, never the sole encoding, one hue one meaning.
 Draft
Numbers, units, and empty states
Format the number for scanning, state the unit once, carry the denominator — and give no-data, real-zero, suppressed and loading four distinct treatments so a blank never reads as zero.
 Draft
Charts at tile scale
A tile is not a shrunk panel: drop the axis, gridlines and legend; use a big number and delta, a bullet bar, direct-labelled bars, or a sparkline — a chart that needs an axis needs a panel.
 Draft
Reading the chart honestly
The distortions review keeps re-litigating, settled: truncate only position marks not bars, no dual axes, points vs percent, raw vs risk-adjusted, suppress tiny-N rates, and don't cherry-pick the window.
