Charts at tile scale

A dashboard tile is not a shrunk chart panel. At tile size the chrome — axes, gridlines, legends — costs more than it earns, so drop it and keep the mark. A two-bar comparison is usually a big number and a delta; below a legibility floor, the honest chart is a number.

Our context

VC dashboards pack many measures into a tile grid on large desktop monitors — the overview an analyst or care manager scans before opening anything. Each tile gets seconds and a few square inches. The deciding constraint is a tile answers one question at a glance; anything that doesn't serve that one answer is noise the tile can't afford.

The rule

At tile scale, drop the chrome and keep the mark: no axis, no gridlines, no legend. Show one value as a big number with a delta, a target as a bullet bar, two or three categories as direct-labelled bars, and a trend's shape as a sparkline. Below the point where labels stop fitting, the tile is a number — and a chart that needs an axis needs a full panel, not a tile.

01 / Quick reference

What the tile has to say picks the form

Start from the one thing the tile must answer, then take the smallest form that answers it without an axis or a legend. If none of these fit, it isn't a tile.

The tile answers Form Why
One value, and how it moved A big number with a delta; a sparkline if the shape matters. The number is the answer; the delta is the context.
One value against a target A bullet bar — a fill plus a target tick. Position against the tick reads in one glance.
Two or three categories Direct-labelled bars — no axis, no legend. The label on the bar replaces the whole scale.
The shape of a trend, not the values A sparkline. Direction and volatility, no room spent on ticks.
Anything more Not a tile — open a panel. If it needs an axis to be read, it needs the width.

03 / The reasoning

Why the tile sheds its chrome

A tile has a few square inches and a few seconds. That budget goes to the answer, so everything that isn't the answer has to justify itself — and at this size, most chart chrome can't.

1

Drop the chrome, keep the mark

Axes, gridlines and legends are navigation for a large chart. At tile size they shrink below legibility and eat the space the mark needs. Remove them; let a direct label or a single number carry the scale.

2

A number often beats a chart

A two-value comparison is usually a value and a delta. "1,284, up 6.2%" answers faster than two bars with an axis. Reserve marks for shape — a trend, a distribution, a gap against a target.

3

There is a legibility floor

Below the size where labels fit, a chart stops informing and starts decorating. When the mark can no longer be labelled or read, promote the value to a number, or the chart to a full panel.

4

A dashboard is small multiples, not shrunk panels

Build the grid from tiles that each answer one question, not from analytical panels crushed to fit. The data canvas spreads to fill; the tile packs — the same width rule from the breakpoints record.

04 / What the field says

The dashboard tradition is clear on this

The people who defined dashboard design say the same thing: at a glance, strip the chrome, and use the number or the sparkline. The tile is a summary, and a summary that needs an axis isn't one.

Stephen Few · Information Dashboard Design

A dashboard is a glance medium

Few's core argument: a dashboard must be read at a glance, so it strips non-data ink, uses compact forms — sparklines, bullet graphs — and shows a single, well-chosen measure per tile. Sparklines and bullet graphs in this record are his.

glance-first, minimal chrome
Edward Tufte · sparklines

A word-sized graphic

Tufte defined the sparkline as a small, high-resolution graphic with no axis or gridlines — the shape of a series in the space of a word. It is exactly the trend form a tile can afford.

axis-free trend shape
IBM · Carbon — KPI & expressive

Big value, then a small trend

Carbon's KPI patterns lead with the value and a delta and add a compact trend only in support — no axis furniture at summary size. It is the enterprise version of the big-number tile.

value + delta + small trend
Google · Material — data & scorecards

Scorecards summarise a single metric

Material's scorecard is a single metric with its change, sized for scanning, distinct from the full charts it links to. Same split VC needs between the tile and the panel behind it.

one metric, made to scan
VC · Breakpoints

Width follows content type

The breakpoints record already sets the frame: data canvases spread to fill, forms cap to a measure. Tiles are the packed end of that — many small summaries, not one panel shrunk to fit.

tile packs, canvas spreads
Synthesis

The pattern for tools like ours

For a VC overview grid on a large monitor: one question per tile, in its smallest honest form — number, delta, sparkline, direct-labelled bars, bullet — and no axis. If it needs a scale to be read, it needs a panel. Our usage should follow it.
